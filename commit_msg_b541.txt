BUILD 541: fix 0x139 BSOD when new process inserted during DKOM hide

UnhideProcess list-integrity check + self-loop fallback.
Root cause: HideProcess makes prev to next (current unlinked). If a new
process C is created and inserted between prev and next during hide,
original UnhideProcess overwrites C list entry so C becomes dangling
and C exit triggers RemoveEntryList check failure then BugCheck 0x139.
Fix: read prev Flink and next Blink current values; if unmodified
use original restore; if modified (new process inserted) use self-loop
(current Flink = current, current Blink = current) without touching
prev or next, passing RemoveEntryList debug check without breaking C.
