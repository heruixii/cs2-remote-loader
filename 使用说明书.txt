一、是什么
================================================================

一个 CS2 ESP 透视外挂。原理：先下载加密的作弊 DLL → 内存中解密
→ 手动注入到 CS2 → 不落盘不留文件。附带 12 项 PAC 规避。

ESP 8 项功能（全部生效）:
  方框 | 血条 | 玩家名 | 距离 | 武器名 | 头部圆点 | 额外信息 | 十字准星

适用平台:
  完美平台 (PAC) — 12项规避策略专门针对 PAC 内核驱动


二、用户端文件
================================================================

只需要 1 个文件:

  loader.exe     — 右键管理员运行即可，无需其他文件

（不需要 libwinpthread-1.dll、不需要 payload.dat、不需要 DLL）


三、前置条件（必读！）
================================================================

  【1】关闭 Windows 内核隔离 (HVCI)
       → Windows 安全中心 → 设备安全性 → 内核隔离 → 关闭
       → 不关闭无法加载 BYOVD 内核驱动，PAC 防御全部失效

  【2】已安装完美世界对战平台
       → 平台自动安装 PAC (MessageTransfer.sys)


四、使用方法（必读！）
================================================================

  【第1步】正常启动完美世界对战平台（自动加载 PAC）

  【第2步】通过平台启动 CS2，进入对局地图（练习/休闲/竞技都可以）
          ⚠️ 不要在大厅就运行！必须进去能看到画面！

  【第3步】右键 loader.exe → 以管理员身份运行

  【第4步】等待 ~5~10 秒，基础.exe 窗口弹出

  【第5步】切回游戏，屏幕中央应该有绿色十字准星，
          敌人身上有红色方框 + 血条 + 名字

  一句话：先开平台进地图，再右键管理员运行 loader。


五、错误启动时机及风险
================================================================

  错误时机                    后果                          风险
  ─────────────────────────────────────────────────────────────────
  完美平台没开                 PAC 未安装，防御层空转（但无实际风险）  低

  在大厅运行（还没进地图）     注入成功但没有EntityList数据，   中
                             ESP空白

  先开loader，后开CS2         找不到cs2.exe，直接退出         低

  CS2还没开                   同上                           低

  没右键管理员运行             BYOVD驱动加载失败，            极高
                             PAC内核回调摘除不生效
                             → 基础.exe OpenProcess 被监控

  系统未重启 (蓝屏后)          RTCore64.sys 设备残留，        极高
                             驱动加载失败 → 防御失效

  HVCI 未关闭                 BYOVD 驱动被 Windows 阻止      高

  同时开着其他反作弊游戏       EkkoSleep内存加密+ETW patch    高
                             可能被别的AC检测

  VPN/网络断开                无法下载payload.dat             低


六、如何确认生效
================================================================

  正常效果:
    - 屏幕中央显示绿色十字准星（有没有敌人都显示）
    - 敌人身上: 红色方框 + 绿色血条 + 名字 + 距离 + 武器名

  如果 30 秒后还看不到:
    1. 确认是否在游戏地图内（不是大厅）
    2. 确认是否管理员运行
    3. 查看 %TEMP%\stealth_diag.log 找报错:
       ✅ "BYOVD: SUCCESS with RTCore64.sys"          — 驱动加载成功
       ✅ "callbacks removed (PAC/MessageTransfer)"   — 回调摘除成功
       ✅ "NEUTRALIZING MessageTransfer"              — minifilter 中和成功
       ❌ "ZOMBIE DEVICE DETECTED"                    — 需重启系统
       ❌ "ALL N drivers failed"                      — 驱动加载失败


七、注意事项
================================================================

  - loader.exe 运行后自动删除自身，请保留备份再测试
  - payload 在 CS2 进程内存中存活，关闭 CS2 即自动清理
  - 窗口最小化后 overlay 可能浮在桌面，切回游戏即正常
  - 改变游戏分辨率后 overlay 不会自动适配，需重启
  - 测试请用小号！不要在主力号上用
  - 蓝屏后必须重启系统清除 RTCore64 残留


八、PAC 防御原理（简化版）
================================================================

  我们的 BYOVD 驱动防御针对 PAC 做了这 3 件事:

  ① 摘除 PAC 内核回调
     → PAC 注册的 ObRegisterCallbacks（句柄监控）/ ProcessNotify
        / ImageNotify 全部被 NULL 化 → PAC 内核组件"失明"

  ② 中和 PAC minifilter
     → MessageTransfer.sys 是文件系统过滤驱动，会拦截所有文件操作
     → 我们把它的回调全部替换为空函数 → 文件扫描失效

  ③ 清理驱动痕迹
     → 清除 MmUnloadedDrivers / PiDDBCacheTable / KernelHashBucketList
       中的 RTCore64 记录 → PAC 无法发现我们加载过漏洞驱动


九、完美平台安全建议
================================================================

  PAC 除了内核驱动检测，还有服务端行为分析:
    - 别隔着墙预瞄人
    - 别每局杀太多（保持 KD 合理）
    - 别太明显（走路预瞄在墙后敌人位置）
    - 适当故意"失误"，维持行为自然

  PAC 目前无内核心跳机制，我们的防御已覆盖已知所有内核检测路径。
  如果平台更新，重新编译最新版 loader 即可。


十、故障排查
================================================================

  现象                              原因及解决
  ─────────────────────────────────────────────────────────
  loader 双击闪退                  网络问题/URL失效，看控制台报错

  管理员运行后没任何效果            - 没进地图（在大厅运行了）
                                   - 不是管理员运行
                                   - 完美平台没开
                                   - 防火墙拦截了GitHub下载

  CS2闪退/崩溃                     运行时机不对，先进地图再运行

  完美平台提示"检测到异常环境"      在安全检测期间运行了loader，
                                   等检测完再运行

  蓝屏 (BSOD)                      系统驱动被误伤（已修复），
                                   重启后使用最新版

  杀软拦截 loader.exe              误报，加白名单


十一、管理端（你的操作）
================================================================

  1. 修改 loader.cpp → PAYLOAD_URLS[]，换成你的服务器URL
  2. 可选修改 XTEA_KEY[4]（encrypt.cpp 和 loader.cpp 同步改）
  3. 运行 build.bat 编译
  4. 把 payload.dat 上传到你的HTTP服务器
  5. 分发 loader.exe 给用户

  用户端只需要 loader.exe 一个文件，不需要其他。


十二、安全提示
================================================================

  1. 测试用小号
  2. 不要隔墙瞄人，不要每局杀太多
  3. 每次重新编译换 XTEA 密钥更安全
  4. 此工具仅供学习研究
