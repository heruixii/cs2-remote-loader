BUILD 531: test mode skip StealthSleep (EkkoSleep) to fix self-encryption crash

测试2 主循环第一帧 ReopenProcessHandle 之后崩溃, 无 CRASH 日志.
根因: EkkoSleep 的 EncryptAll 加密了自身代码页.

注册逻辑 (payload.cpp L1075-L1105) 只豁免 2 个页面:
- ekkoPage (GetSelfPage = EkkoSleepPageMarker 所在页)
- vehPage (DiagVehHandler 所在页)

但 EncryptAll (memory_cloak.cpp L134) 和 DecryptAll (L160) 在
EkkoSleepPageMarker (L56) 和 EkkoSleep (L208) 之间, 编译后可能在
ekkoPage 之前的页, 被注册并加密. EncryptAll 加密自身代码页后返回,
执行已加密代码 → 进程崩溃 (无 CRASH 日志, 因 DiagLog 所在页也被加密).

修复: 测试模式跳过 StealthSleep, 用普通 Sleep(sleepMs) 代替.
测试模式无 CS2 无反作弊扫描, 不需要内存加密.
正常模式 (测试3) 仍需 EkkoSleep, 后续需修复豁免逻辑.

3次代码审查通过:
1. 正确性 (条件判断/大括号/Sleep API 安全)
2. 影响范围 (测试模式不影响 E+G 其他组件, 正常模式不受影响)
3. 崩溃风险 (消除 EncryptAll 自加密崩溃, Sleep 简单 API 不崩溃)
